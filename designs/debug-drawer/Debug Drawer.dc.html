SloopWorks · ADR 0008 design gate

Debug Drawer — hi-fi mockups

A reusable, debug-only developer instrument for Compose Multiplatform: one floating bubble opens a host drawer of pluggable panels. Every screen and state below is shown in the default SloopWorks skin and a consumer-themed Dayfold instance, light and dark — proving the theming contract. Architecture is fixed; this gate is for the visuals and interaction. Full written spec in spec.md.

INTERACTION

Tap the bubble → scrim fades in (120ms) and the sheet springs from the dock edge (180ms, ease-out). The bubble is the app's entire debug footprint and is absent from release builds — its absence is the signal. ≥48dp target; draggable to any corner; snaps to the nearest edge on release.

PANEL NAV · LIST → DETAIL

The host opens to this list. Every row is a registered panel: leading icon, title, a live trailing value or status, chevron. Rows are ≥64dp tall (≥48dp target + padding). Tapping pushes a detail view; the header swaps to a back affordance. Status colours (the red 3 E) are fixed across every app for cross-app legibility.

ADAPTER COMPROMISE (flagged)

The Redux inspector is a rich third-party surface forced into the host's list→detail chrome. We give it a full-height pane and let it own its internal scroll & time-travel slider, but its language won't be 100% native. This is the architecture's one real UX trade — noted in the spec, not redesigned.
