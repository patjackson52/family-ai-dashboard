IAccent rule · data fields

The rule, and the fields

paletteAccent-color usage rule

A hub/card's accentColor is a seed, not a paint. The renderer harmonizes it — keep hue, clamp chroma, pin lightness into a legible band per theme — then applies the derived roles only to decorative surfaces: the card left-edge/border, the icon tile, chips, and the hero scrim tint. It is never placed on body text, headlines, or large background fills, where an arbitrary brand hex would fail WCAG 1.4.1 / contrast. Meaning is always carried by icon + text; color only reinforces. Loading, error and "nothing set" resolve to the same accent tile so a missing image is invisible.

data_objectFields used — & one flag
Built entirely on the §6 model — no new fields invented:
Hub {heroUrl, thumbnailUrl,
  heroFit: cover|contain, imageAlt,
  icon, accentColor}
Card {icon, accentColor, thumbnailUrl,
  imageAlt, imageFit}
block {thumbnailUrl | avatarUrl}
flagFlag: icon should be constrained to the curated named set (enum), not a free string — so an unknown name falls to the plain default rather than rendering nothing. No schema change, just validation.
