# Hub Sync Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give the Hubs surface (shipped #42, plain GET, no cache) offline incremental sync — instant cold-start render, foreground freshness, and ADR-0030 revocation — by extending the existing card-sync machinery.

**Architecture:** Hubs ride the **existing** `GET /families/:fid/sync` envelope as additional entity types in one **merged-typed keyset** stream (cursor `base64(updated_at|type|id)`), the **existing** `ContentStore` (one SQLDelight DB, one cursor, one `wipe()`), and a **second** `SyncEngine` DB→store bridge. No new `HubStore`/`HubClient`/`HubEngine`. Ships in two PRs: **PR1 = hubs-list offline + revocation** (no DB migration needed — the shipped `0009` trigger already bumps `hubs.updated_at`); **PR2 = hub detail tree** (sections/blocks, the security-critical hub-visibility join + the `0010` child fan-out).

**Tech Stack:** TS / Hono / Postgres (api) · vitest · Kotlin Multiplatform / Compose / redux-kotlin (alpha) · SQLDelight 2.3.2 · JDK 17.

## Global Constraints

- **JDK 17** for all Gradle: `JAVA_HOME=/opt/homebrew/Cellar/openjdk@17/17.0.18/libexec/openjdk.jdk/Contents/Home`. Client build from `apps/`: `./gradlew :client:desktopTest`. API from `apps/api`: `DATABASE_URL=postgres:///fad_test npx vitest run`.
- **Pure reducer** (ADR 0013): all I/O in `SyncEngine`; reducer arms are pure full-replaces of disjoint slices. **Network never touches the store directly** — only the DB flow dispatches `HubsLoaded`/`HubTreeLoaded`.
- **DB-as-truth** (ADR 0020): the SQLDelight DB is the single source; the store is a projection via a reactive flow. The cursor lives in `sync_meta`, never redux.
- **Keyset comparison is row-wise**: `(updated_at, type, id) > ($1,$2,$3)` — never hand-expanded AND/OR (the classic keyset off-by-one). Mirror `apps/api/src/repo.ts` card keyset.
- **One writer per store slice**: `state.cards` ← `CardsLoaded` only; `state.hubs` ← `HubsLoaded` only; `state.currentHubTree` ← `HubTreeLoaded` only. Delete any network-fed dispatch of these.
- **Verify test COUNTS rise**, not just BUILD SUCCESSFUL (the JUnit non-Unit-test gotcha — use `runBlocking<Unit>` where a test's last expr isn't Unit).
- **Migration numbering**: api migrations `0001`–`0009` exist; next free is `0010`. SQLDelight client migration: next free `.sqm` is `1.sqm` (migrates schema v1→v2).
- **No new visible surface**: `HubListScreen`/`HubDetailScreen` render byte-identically (DB-fed parity is a DoD snapshot test). ADR-0008 design gate already cleared (INB-22).

- [ ] **Step 3: Implement `syncContent` in `repo.ts`**

Add a merged keyset over a typed UNION. Mirror the existing `syncCards` row-wise keyset; `type` sits between `updated_at` and `id` so the tuple is globally unique.

```ts
// repo.ts — merged keyset for cards + hubs (PR2 adds section/block UNION arms).
export async function syncContent(
  familyId: string, su: string, si: string, type: string, limit: number,
) {
  const r = await q(
    `SELECT updated_at, type, id, family_id, deleted_at, payload FROM (
       SELECT updated_at, 'card' AS type, id, family_id, deleted_at,
              to_jsonb(briefing_cards.*) AS payload FROM briefing_cards WHERE family_id=$1
       UNION ALL
       SELECT updated_at, 'hub' AS type, id, family_id, deleted_at,
              to_jsonb(hubs.*) AS payload FROM hubs WHERE family_id=$1
     ) merged
     WHERE (updated_at, type, id) > ($2::timestamptz, $3, $4)
     ORDER BY updated_at, type, id LIMIT $5`,
    [familyId, su === "" ? "-infinity" : su, type, si, limit],
  );
  return r.rows as { updated_at: string; type: string; id: string; deleted_at: string | null; payload: any }[];
}
```

- [ ] **Step 4: Wire the `/sync` route to the merged path with cursor codec**

In `app.ts` `/families/:fid/sync`: decode the cursor by part-count (2 = legacy cards-only via `syncCards`; 3 = merged; empty/bad = full resync from `-infinity`). For each merged row compute visibility per type (`cardVisible` for cards, `hubVisible` for hubs); **live** if visible+not-deleted, else **tombstone** `{type, id}`. Encode `next_cursor = base64("${last.updated_at}|${last.type}|${last.id}")`; `has_more = rows.length >= SYNC_LIMIT`. Validate the decoded `type ∈ {card,hub}` (enum — unknown → full resync; never inject into the WHERE).

```ts
const raw = c.req.query("since") ?? "";
let su = "", st = "", si = "";   // start updated_at / type / id
if (raw) {
  const parts = Buffer.from(raw, "base64").toString().split("|");
  if (parts.length === 3 && ["card","hub"].includes(parts[1]) && !Number.isNaN(Date.parse(parts[0]))) {
    [su, st, si] = parts;
  } else if (parts.length === 2 && !Number.isNaN(Date.parse(parts[0]))) {
    return legacyCardSync(c, fid, caller, parts);   // existing syncCards path, 2-part cursor
  } // else: leave su/st/si empty → full resync
}
const rows = await repo.syncContent(fid, su, st, si, SYNC_LIMIT);
const allow = new Map<string, Set<string>>(); // hubId -> allow-list, lazy per visible hub
const changes = { cards: [] as any[], hubs: [] as any[] };
const tombstones: { type: string; id: string }[] = [];
for (const r of rows) {
  const visible = r.deleted_at ? false :
    r.type === "card" ? cardVisible(r.payload, caller)
    : hubVisible(r.payload, caller, (hid) => (allow.get(hid) ?? new Set()).has(caller.userId ?? ""));
  // (for hubs, prefetch allow-lists for the page's restricted hubs before the loop — see note)
  if (visible) (r.type === "card" ? changes.cards : changes.hubs).push(r.payload);
  else tombstones.push({ type: r.type, id: r.id });
}
const last = rows[rows.length - 1];
const next_cursor = last ? Buffer.from(`${last.updated_at}|${last.type}|${last.id}`).toString("base64") : raw;
return c.json({ changes, tombstones, next_cursor, has_more: rows.length >= SYNC_LIMIT });
```

Note: for the page's `restricted` hubs, prefetch their `resource_visibility` allow-lists in one query before the loop (mirror the single-hub `allowListFor` at `app.ts:433`), keyed into `allow`.

```sql
-- Hub-sync cache (PR1). Mirrors `card`: business cols + JSON-TEXT cols decoded at projection + deleted flag.
CREATE TABLE hub (
  id          TEXT NOT NULL PRIMARY KEY,
  type        TEXT NOT NULL,
  title       TEXT NOT NULL,
  status      TEXT,
  start_at    TEXT,
  end_at      TEXT,
  countdown_to TEXT,
  visibility  TEXT,
  updated_at  TEXT NOT NULL,
  deleted     INTEGER NOT NULL DEFAULT 0
);

upsertHub:
INSERT INTO hub(id, type, title, status, start_at, end_at, countdown_to, visibility, updated_at, deleted)
VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, 0)
ON CONFLICT(id) DO UPDATE SET
  type=excluded.type, title=excluded.title, status=excluded.status,
  start_at=excluded.start_at, end_at=excluded.end_at, countdown_to=excluded.countdown_to,
  visibility=excluded.visibility, updated_at=excluded.updated_at, deleted=0;

markHubDeleted:
UPDATE hub SET deleted=1, updated_at=? WHERE id=?;

activeHubs:
SELECT id, type, title, status, start_at, end_at, countdown_to, visibility
FROM hub WHERE deleted=0 ORDER BY COALESCE(start_at, countdown_to), id;

wipeHubs:
DELETE FROM hub;
```

- [ ] **Step 1: Extend `applyDelta` + add hub projection + extend `wipe`** (one atomic txn)

```kotlin
fun applyDelta(
  changedCards: List<Card>, changedHubs: List<Hub>,
  tombstones: List<Tombstone>, nextCursor: String?, nowIso: String,
) {
  q.transaction {
    changedCards.forEach { c -> q.upsertCard(/* …existing 14 args… */, nowIso) }
    changedHubs.forEach { h ->
      q.upsertHub(h.id, h.type, h.title, h.status, h.startAt, h.endAt, h.countdownTo, h.visibility, nowIso)
    }
    tombstones.forEach { t ->                       // total over types; unknown ignored
      when (t.type) { "card" -> q.markDeleted(nowIso, t.id); "hub" -> q.markHubDeleted(nowIso, t.id) }
    }
    if (nextCursor != null) q.setCursor(nextCursor, nowIso)
  }
}

fun activeHubsFlow(): Flow<List<Hub>> =
  q.activeHubs().asFlow().mapToList(Dispatchers.Default).map { rows -> rows.map(::rowToHub) }

private fun rowToHub(r: com.sloopworks.dayfold.client.db.ActiveHubs): Hub = Hub(
  id = r.id, type = r.type, title = r.title, status = r.status,
  startAt = r.start_at, endAt = r.end_at, countdownTo = r.countdown_to, visibility = r.visibility,
)

fun wipe() { q.transaction { q.wipeCards(); q.wipeHubs(); q.wipeCursor() } }
```

- [ ] **Step 1: Write the migration** (extend the shipped `touch_hub_from_visibility` so a `resource_visibility` change re-surfaces the hub's sections + blocks too)

```sql
-- 0010_hub_sync_fanout.sql — ADR 0030 revocation for the hub SUBTREE.
CREATE OR REPLACE FUNCTION touch_hub_from_visibility() RETURNS trigger AS $$
DECLARE h text := COALESCE(NEW.hub_id, OLD.hub_id);
        f text := COALESCE(NEW.family_id, OLD.family_id);
BEGIN
  UPDATE hubs     SET updated_at = now() WHERE family_id = f AND id = h;
  UPDATE sections SET updated_at = now() WHERE family_id = f AND hub_id = h;
  UPDATE blocks   SET updated_at = now() WHERE family_id = f AND section_id IN
         (SELECT id FROM sections WHERE family_id = f AND hub_id = h);
  RETURN NULL;
END $$ LANGUAGE plpgsql;
-- trigger trg_resvis_touch_hub (0009) already binds this function on resource_visibility.
```
